# bfc 语言与工具链规范（V1 草案）

状态：草案（面向学习的编译器配置）

## 1. 范围与非目标

本文档定义 `bfc` 的第一个稳定配置（profile）。

目标：
- 定义一个确定性的语言/运行时行为配置。
- 支持多文件输入，并在前端合并为单一编译单元。
- 从合并后的输入中产出可执行文件或库产物。

V1 非目标：
- 完整工业级链接器特性。
- 在源码中由用户手写原始地址。
- 语言核心中的函数调用语法与调用约定。
- 动态库输出的完整实现（仅保留 TODO 接口）。

## 2. 输入模型

`bfc` 接受一个或多个源文件。

每个源文件可以包含：
- `plt` 段（符号声明/导入）。
- `text` 段（函数定义）。

禁止在源码层手写地址。符号通过名称识别；最终地址由目标文件格式/链接器分配。

## 3. 合并模型（前端链接）

在代码生成前，所有输入文件会被合并为一个全局编译单元。

### 3.1 符号规则

- 同名函数重复定义：编译错误。
- 重复声明且签名不兼容：编译错误。
- 重复声明且签名一致：允许（去重）。
- 引用了未定义符号：编译错误。

### 3.2 诊断

诊断信息在可用时必须包含文件与位置信息。

## 4. 入口点与产物选择

入口点通过函数定义上的 `@entry` 注解识别。

合并后的规则：
- 入口点数量为 `0`：按库进行编译。
- 入口点数量为 `1`：按可执行文件进行编译。
- 入口点数量大于 `1`：编译错误。

不要求硬编码函数名（如 `main`）。

## 5. 产物策略

V1 输出策略：
- 库场景（`0` 个入口）：只输出静态库（`.a`）。
- 动态库不属于 V1。若用户请求动态库，编译器必须报 `E_ARTIFACT_KIND_UNSUPPORTED`。
- 可执行场景（`1` 个入口）：输出可执行二进制文件。

## 6. 核心语言语义

## 6.1 Cell 类型与算术

V1 固定 cell 语义：
- Cell 类型：有符号 64 位整数（`i64`）。
- 算术规则：二补码回绕语义（按 `2^64` 位模式进行模回绕）。
- 在该配置中，算术溢出不是未定义行为，也不会触发运行时错误。

含义：`Overflow` 诊断属于未来 checked 配置的扩展点，不是 V1 的激活运行时行为。

## 6.2 Tape 与指针

V1 的 tape 模型固定为：
- tape 长度固定为 `30_000` 个格子。
- 指针初始位于 cell 索引 `0`。
- 指针移动到区间 `[0, 29_999]` 之外时，必须在运行时触发 `E_PTR_OUT_OF_BOUNDS` 并以非零状态终止执行。

## 6.3 I/O（`','` 与 `'.'`）

`','` 与 `'.'` 是流操作，不是函数参数/返回值机制。

V1 I/O 行为固定为字节流：
- 输入流：`stdin`。
- 输出流：`stdout`。
- `,`：从 `stdin` 读取 1 字节。
	- 读取成功：将该字节值（`0..255`）写入当前 cell（作为 `i64`）。
	- EOF：将 `0` 写入当前 cell。
- `.`：将当前 cell 的低 8 位（`cell as u8`）输出到 `stdout`。

V1 不定义 UTF-8 标量输出模式。

## 7. 函数调用

V1 不支持源码层函数调用。

- V1 不定义 `call` 关键字或等价调用指令。
- 任何类调用语法必须报编译错误 `E_CALL_UNSUPPORTED`。

## 8. 建议的 CLI 契约

示例：

```bash
bfc a.bf b.bf -o out
```

行为：
- 解析并合并所有文件。
- 校验符号与入口点约束。
- 根据入口点数量选择产物类型。
- 从合并后的编译单元生成一份对象管线输出。

## 9. 扩展点（V1 之后）

- 附加 cell 配置（例如 `u8` 回绕、checked 算术）。
- 替代 tape 策略（例如自动扩展 tape）。
- 显式 I/O 模式开关（包括 UTF-8 标量输出模式）。
- 函数调用语法与 ABI。
- 动态库输出。

## 10. 取舍依据

这些 V1 决策在“实现难度”与“主流低层编译器常见方案”之间做了偏保守且可快速落地的取舍：

- 固定长度 tape（`30_000`）+ 显式越界 trap：运行时模型简单且行为可复现。
- 字节流 I/O（`stdin`/`stdout`，输出低 8 位）：后端最容易映射到常见 C/OS 原语。
- i64 回绕算术：语义明确、无 UB、代码生成简单。
- 前端合并模型：V1 避免实现完整链接器，但仍可学习符号与入口诊断。
